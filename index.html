Heavy Vehicles Detection

Heavy Vehicles Detection using RCNN


About the Project

  • We present a classification and detection system of heavy vehicles based on computer vision techniques. The system is designed to automatically gather important statistics for policy makers and regulators in an automated fashion. These statistics include vehicle counting, vehicle type classification. The core of such system is the detection and classification of heavy vehicles in traffic videos. We implement two models for this purpose, first is a MoG & SVM system and the second is based on Faster RCNN, a recently popular deep learning architecture for detection of objects in images.
  • We show in our experiments that Faster RCNN outperforms MoG in detection of vehicles that are static, overlapping or in night time conditions. Faster RCNN also outperforms SVM for the task of classifying vehicle types based on appearances. In the proposed heavy vehicle detection and counting system, the road surface in the image is first extracted and divided into a remote area and a proximal area by a newly proposed segmentation method; the method is crucial for improving vehicle detection. Then, the above two areas are placed into the YOLOv3 network to detect the type of the vehicle.
  • Finally, the vehicle trajectories are obtained by the ORB algorithm, which can be used to judge the driving direction of the vehicle and obtain the number of different vehicles. The experimental results verify that using the proposed segmentation method can provide higher detection accuracy, especially for the detection of Heavy vehicle objects. Moreover, the novel strategy described in this article performs notably well in judging driving direction and counting vehicles.

Drone Detection

Drone Detection


About the Project

Introduction
This project aims to develop a drone detection model using YOLO (You Only Look Once) architecture and a dataset from Roboflow. The goal is to create an efficient and accurate system capable of detecting and identifying foreign drones attempting to enter our country's borders.
Dataset Collection and Preprocessing
We Utilized a drone dataset from Roboflow, which contains annotated images of drones and background scenes. 		Performed dataset preprocessing steps, including data augmentation techniques such as rotation, scaling, and flipping to increase variability.
We then split the dataset into training, validation, and testing sets, ensuring a representative distribution of drone and background images.
Training
We implement the YOLO-NAS architecture, which is a state-of-the-art object detection algorithm,

to train a model using the annotated drone dataset to adapt it to the specific detection task.
Border Security System
Developed a real-time drone detection system that takes live video feed or image streams. Apply the trained YOLO model to each frame or image to detect and localize drones. We plan to integrate the detection results into the border security system, triggering alerts or initiating appropriate response actions upon drone detection.
Digital Warehouse

Digital Warehouse


About the Project

Introduction
This project aims to develop a warehouse management system that utilizes CCTV footage to automate goods counting using computer vision techniques. CCTV Footage Collection Install CCTV cameras strategically in the warehouse to capture footage of the goods storage areas. Ensure the cameras have suitable angles and resolutions for effective object detection.
Footage Preprocessing
Extract frames from the CCTV footage to obtain individual images for analysis. Perform necessary preprocessing steps such as resizing, noise reduction, and illumination adjustment to enhance image quality.
Object Detection and Counting
Utilize object detection algorithms, such as YOLO (You Only Look Once), to identify goods in the captured images.		 Then count the goods using the common shapes that the goods are packed in. Using basic feature detection algorithms (e.g circle Hough Transform) available in OpenCV we were able to count the goods with little error.
Calibration and Calibration Validation
Calibrate the system by determining the conversion factor between the number of pixels and the physical size of the goods. Validate the accuracy of the calibration by manually counting a subset of goods and comparing the results with the system's count.
Real-time Goods Count Management
Integrate the developed algorithm into a real-time system that continuously processes the CCTV footage. Apply the object detection, tracking, and counting techniques to update the goods count in database.
Performance Evaluation
Assess the system's accuracy, precision, and recall by comparing the automated goods count with manual counts. Measure and analyze the system's processing speed and resource utilization to ensure it meets real-time requirements.
Limitations
The detection and counting might be hindered by a poorly lit room. There is always a certain degree of error that can occur when counting the goods. Difficulties might be faced when counting goods that are placed behind one another.
Conclusion
A digitalized warehouse management system will eliminate human error and reduce employment costs, as with this system, fewer employees will be required to manage the warehouse.

BARASHIGRI glacier

BARASHIGRI GLACIER MELTING TRACKING SYSTEM


About the Project

Introduction
This project investigates the relationship between various greenhouse gas emissions and global temperature changes using multiple linear regression analysis. The study utilises data from a range of sources, including CO2 emissions, NO2 emissions, SO2 emissions, methane emissions, and global warming emission rate parameters.
Study area
The Bara-Shigri glacier is one of the largest glaciers located in the Chandra Valley, Himalayas, India. It is around 28 km long and has an area of 126.45 sq km and possesses numerous tributary glaciers. The glacier is flowing northwards and debouches into the Chenab River. The average altitude of the glacier is 4200 metres.

By providing this information, the methodology section of the documentation can help readers understand the rigour and validity of the research and how the data was collected		 and analysed to generate insights into the impact of emissions on climate change.

Ice mass balance
Ice mass balance refers to the net gain or loss of ice over a given period of time. It is typically measured as the difference between the amount of snow and ice that accumulates on a glacier (known as the accumulation zone) and the amount of ice that melts or evaporates over the same period of time.
Glacier retreat
Glacier retreat refers to the shrinking of a glacier's terminus (the end point where the glacier meets land or water) over a given period of time.		 This can occur due to a reduction in ice mass balance, which causes the glacier to become shorter and narrower.

Underwater Archaeology

Underwater Archaeology Using YOLO-NAS


About the Project

Introduction
YOLO-NAS is a new foundational model for object detection that has been shown to be effective for underwater object detection. It is able to learn to identify objects in underwater images even in the presence of significant challenges, such as reduced visibility and color.
Dataset Collection and Augmentation
The Brackish Underwater Object Detection Dataset is a publicly available dataset of underwater images with bounding box annotations of fish, crabs, and other marine organisms. It was collected with a camera mounted 9 meters below the surface on the Limfjords bridge in northern Denmark by Aalborg University.
The dataset contains 14,674 images (12,444 of which contain objects of interest with bounding box annotations) of fish, crabs, and other marine animals. 		The images were captured in a variety of conditions, including different water depths, turbidity levels, and lighting conditions. The dataset is divided into three sets: train, validation, and test. The train set contains 10,792 images, the validation set contains 1,319 images, and the test set contains 2,563 images.
The images in the dataset are labeled with bounding box annotations. The bounding boxes are defined by their top-left corner, bottom-right corner, and width and height.
Model Training and Evaluation
The model will be trained on the training set. The training process will involve iteratively adjusting the model's parameters to minimize the loss function. The loss function measures the difference between the model's predictions and the ground truth labels.


 Model evaluation
The model's performance can be evaluated using a variety of metrics, such as accuracy, precision, and recall. Accuracy measures the fraction of predictions that are correct. Precision measures the fraction of positive predictions that are correct. Recall measures the fraction of positive examples that are correctly identified.
Model Performance
YOLO-NAS models have been shown to achieve state-of-the-art performance on a variety of object detection benchmarks. YOLO-NAS models have achieved the following results
YOLO-NAS (m) - 43.5 mAP
YOLO-NAS (l) - 46.2 mAP
YOLO-NAS (x) - 48.3 mAP
Conclusion
As a result of its superior performance It is used in a variety of applications such as underwater search and rescue, marine conservation, underwater archaeology.
